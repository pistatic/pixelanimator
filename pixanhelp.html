Getting started
Here's a quick way to try out the program:
  • On startup the program starts with an empty 50×60 pixels spritesheet with 12 frames.
  • Click on the bottom timeline to change the current frame. You'll see the red frame moving.
    (As long as you click between 1-12 frames)
  • Now click the "Onion Toggle" button once. This will allow seeing neighbour frames while animating.
  • With the right mouse button you could set which of the neighbour frames you want to see;
    click on the timeline next to the current frame on both sides!
  • Now select the first frame then draw a little cirlce in the window called "new"!
  • Now select the second frame (you could also use the arrow buttons on your keyboard);
    you should see a transparent image of the circle you just drawn.
  • Draw a second circle over the previous one, but move it a bit away in one direction.
  • Select the third frame and draw an another cirlce, now a bit further away from the second one.
  • Continue drawing until you reach the last frame!
  • When you're done press the Play button, or press Enter.
  • Congratulations! You've just made your first animation in PixelAnimator!
    Now try something more complex!
Loading a sprite sheet
If you want to load a spritesheet do the following:
  • Click the "New Spritesheet" button on the top right.
  • In the popup window set the spw value to your spritesheet's width; and the sph to the height!
  • After you've created the new empty sheet click the "Open Spritesheet" button, and load your sprite!
  • Done!

Right Click Erases a 2×2 line. Picks the color under the current cursor position,
and sets it as the shade color.

Important Notes
The program is under development, so expect bugs and lacking features.
Some of the most noteable:
  • No warnig on quit. Save your work often, and do not press Ctrl+W accidentally
  • Opening a file overwrites the currently active one. Make an empty sheet before doing so.
  • The first frame of sheets rarely gets eaten by the undobuffers; haven't found a cause yet;
    save your first frames!
  • Deleting and adding frames doesn't have undo, and overthrows the undo buffer, so be cautious!
  • Opening the same file twice in a row will fail. Open something else, and try again.
  • Many other minor ones, most of them without any serious consequences; I'm working on them though.
About
